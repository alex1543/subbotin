1) Вопрос:
-----------------------
  В файле CP866_out.txt отсутствуют строки из RSS ленты, а другие
присутствуют. И вообще, можно одни строки выводить из леты, а другие нет?

Ответ:

 KR> Теперь с шаблоном news.tpl (http://freestudents.ru/forum/rss.php):
 KR>    │ > Программное обеспечение :: RE: Можно ли читать группы новостей

  Это потому что в шаблоне news.tpl не указаны тэги из rss.php.

Hапример, так:
(дальше идет часть ленты rss.php)

────────────
<item>
<title>Студенческая жизнь :: RE: Бал Первокурсника - 2006</title>
<dc:creator>umka</dc:creator>
<dc:subject>Студенческая жизнь</dc:subject>
<annotate:reference 
rdf:resource="http://freestudents.ru/forum/viewtopic.php?p=59370#59370" />
</item>
────────────

  Теперь шаблон.
  Hапример, тебе надо вывести в файл CP866_out.txt все, что между тэгами
<dc:subject> и </dc:subject>. Тогда надо в news.tpl написать так:

@dc:subject,0,[78]

  Все эти шаблоны именно для этого и нужны, чтобы записывать с CP866_out.txt 
только то, что тебе нужно. А эти глюки появляются из-за того, что в каждом rss 
потоке тэги разные. Универсальный темплейт сделать нельзя.

Как в Windows XP запускать программы как сервисы?

Самый простой способ установки запуска программы в качестве сервиса (службы) - 
использование специальной утилиты-мастера srvinstw.exe из комплекта Windows 
2000 Server Resource Kit. Эта утилита позволяет без труда как устанавливать 
новые сервисы, так и удалять ненужные.

Кроме этого, в Windows 2000 Server Resource Kit входят специально для этого 
предназначенные консольные программы instsrv.exe и srvany.exe. Для работы с 
ними нужно будет войти в систему под учетной записью Администратора 
(Administrator), скопировать эти файлы в папку %SystemRoot%\System32, а потом 
выполнить следующую команду:
instsrv RSS2FIDO C:\WINDOWS\system32\srvany.exe
где "new_service" - это любое удобное для вас наименование нового сервиса (как 
раз им и будет обозначаться создаваемый сервис). Обратите внимание на то, что в
том случае, если придуманное имя сервиса содержит пробелы, то его нужно будет 
заключить в кавычки.
После проведения этой "хитроумной" операции в диалоговом окне 'Управление 
компьютером' -> 'Сервисы' появится новый сервис, и нужно будет в его свойствах 
установить способ его запуска - Automatic, если сервис должен запускаться при 
загрузке системы, Manual - если вручную, или Disabled - отключен.
Hо и это еще не все: в разделе реестра, соответствующем имени создаваемого 
сервиса ( HKEY_LOCAL_MACHINE \ SYSTEM \ CurrentControlSet \ Services \ 
NEW_SERVICE \ Parameters ) необходимо создать строковый параметр "Application" 
(без кавычек), значением которого должен быть указан полный путь к исполняемому
файлу программы, которую необходимо запускать как сервис.
